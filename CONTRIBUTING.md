# Contributing to this pilot repository

This repository is the public home of an LDT4SSC pilot. This document explains
how members of the pilot consortium and external collaborators can contribute
to it.

## Who can contribute

Members of the pilot consortium are the primary contributors to this
repository. External collaborators may also contribute when relevant; the
pilot's coordinators decide who has commit access.

## What kinds of contributions are welcome

- **Updates** in the [`updates/`](updates/) folder — structured posts about
  the pilot's progress, milestones, lessons, assets, and events. See
  [`updates/README.md`](updates/README.md) for the format.
- **Documentation** in the [`docs/`](docs/) folder — longer-form material
  about the pilot's architecture, decisions, methodology, and context.
- **Asset descriptions** in the [`assets/`](assets/) folder — pointers to
  shared artefacts produced by the pilot, with descriptions of their purpose
  and current state.
- **Source code** in the [`src/`](src/) folder, if applicable.
- **Data model descriptions or references** in the [`data-models/`](data-models/)
  folder.

## How to contribute

The recommended workflow is:

1. **Open an issue first** for substantial changes, so contributors can discuss
   the approach before work begins. Small fixes (typos, broken links,
   formatting) can go straight to a pull request.
2. **Create a branch** off `main` for your changes.
3. **Make your changes** following the conventions documented in this
   repository (especially the update schema, for `updates/`).
4. **Commit with a clear message** describing what changed and why.
5. **Open a pull request** against `main`. The validation workflow will
   automatically check any updates you have added or modified. Address any
   reported issues.
6. **Request a review** from another consortium member.
7. Once approved and the validation workflow passes, the pull request can
   be merged.

For very small changes, the pilot's coordinators may choose to commit
directly to `main` without a pull request. This is a pilot-internal decision.

## Things to keep in mind

- **This repository is public.** Do not commit confidential, sensitive, or
  embargoed information. See the LDT4SSC Pilot Update Guide for guidance on
  what should and should not be published.
- **Respect copyright and licensing.** Do not commit material from third-party
  sources unless properly attributed and licensed for reuse.
- **Personal data** must comply with GDPR. Do not include photos, names, or
  identifying information of citizens or participants without appropriate
  consent.
- **Updates must conform to the schema.** The validation workflow will catch
  most schema problems automatically. If you are unsure, refer to the worked
  examples in `updates/_examples/`.

## Questions

If you are unsure about anything, contact the pilot's coordinators or the
LDT4SSC Help Desk.
